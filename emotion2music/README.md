# Music Generator AI - Max/MSP Integration

This project is an intelligent music generator that utilizes Large Language Models (LLMs) to compose musical scores in JSON format, transmitting them in real-time to Max/MSP via the OSC protocol.

The system features a modular and decoupled architecture, specifically designed to separate control logic, API communication, state management, and data translation.

## Project Structure

- `main.py` (Main controller, OSC server, and orchestrator)
- `state.py` (Application state container / AppState)
- `llm_manager.py` (API communication engine for LLMs)
- `max_communicator.py` (OSC communication logic for Max/MSP)
- `prompts.py` (Template and prompt generator for LLMs)
- `utils.py` (Mathematical functions and data manipulation)
- `config.py` (Constants, file paths, and model configurations)
- `instrument_info.py` (Database of instrument ranges and clefs)
- `system_instruct_new.txt` (System instruction template)
- `[API_KEYS]` (Text files containing service access keys)

## Initial Setup

### 1. Requirements
Install the necessary dependencies using pip:
`pip install ollama anthropic google-genai groq openai python-osc json-repair openpyxl`

### 2. API Key Configuration
The system is configured to read API keys from text files located in the root directory. Create the following files based on the backends you intend to use:

- `api_key.txt` (Gemini)
- `api_key_groq.txt` (Groq)
- `api_key_github.txt` (GitHub Models)
- `api_key_claude.txt` (Anthropic)
- `api_key_openrouter.txt` (OpenRouter)

Insert the raw API key string into the corresponding file (ensure there are no quotation marks or trailing spaces).

## Module Responsibilities

* **`main.py` (Controller):** The entry point. It manages the OSC server (port 8001), receives commands from Max/MSP, and coordinates the interaction between the AI engine (`MusicLLM`) and the output module (`MaxMSPCommunicator`).
* **`state.py` (State):** The core data container. It holds all dynamic variables (BPM, number of voices, chat history, score building). By eliminating global variables, it ensures safe and predictable application state management.
* **`llm_manager.py` (AI Engine):** Manages the entire API call pipeline. It initializes clients, handles conversation context, executes generation cycles, and performs refinement and critique phases.
* **`max_communicator.py` (OSC Output):** A dedicated translator. It converts the generated musical JSONs into OSC messages compatible with Max/MSP. It manages sequential transmission, timing, and array formatting (e.g., pitches, durations).
* **`prompts.py` (Prompt Engineering):** The repository for textual logic. It dynamically generates prompts based on current constraints (e.g., specific instrument ranges, BPM, few-shot examples).
* **`utils.py` (Utilities):** Pure mathematical and formatting functions. These functions are stateless and handle duration calculations, note transpositions, and string formatting for OSC.
* **`config.py`:** Static configuration. It manages directory paths (e.g., `experiments/`), file naming conventions, and available model configurations.
* **`strumenti_info.py`:** Domain database. It contains technical specifications (clefs, note ranges, idioms) for every supported instrument.

## Execution

To start the generator, ensure Max/MSP is configured to receive data on port **8000** and send commands to port **8001**, then execute the following command:

`python main.py`

## Experiment Management
Each execution automatically creates a subfolder within the `experiments/` directory. Inside, you will find:
* `[Timestamp].log`: Detailed execution log.
* `[Timestamp].csv`: Performance metrics (latency, refinement cycles).
* `[Timestamp].xlsx`: Excel report for data analysis.
* `[Timestamp]_diaries.txt`: The AI's compositional reasoning and internal thought process.
* `[Timestamp]_jsons.txt`: Raw JSON outputs generated by the models.
